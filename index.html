HTML

  • The head element contains information about the webpage.
  • THe body element represents the visible content show to the user
  • The h1, h2, h3, h4, h5 and h6 elements represent the level of heading a given text block represents
  • The ul, ol, li elements elements represent unordered lists, ordered lists, and list items
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Debugging is one of the most important parts of the development process. Because you only really understand the code while you are writing it, it's really important to leave comments for yourself to be able to review later if/when a bug or issue were to arise. Using GitHub Co-Pilot can really help with learning to write good comments and documentation
  • You don't need to document each bug fix, except for in GitHub where you can use git commit when you fix a bug
  • External CSS Style sheet preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Conversely, Internal style sheet's contains CSS rules for the webpage in the head section of the HTML file, which apply to that page
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git merge: list your branches. a * will appear next to the currently active branch
  • git status: show modified files in working directory, staged for your next commit
